# Style Guide（暫定）

## 前提
- ツールボックス無し（Signal Processing Toolbox / DSP System Toolbox 等は前提にしない）
- 推奨MATLAB：R2020a以降（`exportgraphics` を使用）
- 目安：R2016b以降（暗黙の拡張・string等を前提）

## 命名
- `src/` の関数は `sp_` 接頭辞（例：`sp_dft.m`）
- 章スクリプトは `chapters/chXX_*.m`（または `.mlx`）

## データ形状（原則）
- 信号は **列ベクトル**（`N x 1`）を基本にする（必要に応じて明記）
- 複素数：共役転置 `'` と転置 `.'` を区別する

## 記号の約束（暫定）
- サンプリング周波数：`fs`（Hz）
- サンプリング周期：`Ts = 1/fs`
- サンプル番号：`n = 0,1,...,N-1`
- DFTビン：`k = 0,1,...,N-1`
- 片側/両側スペクトルのスケーリングは付録B（`tex/90_appendix.tex` の付録B）と各章のtipBoxで統一する（振幅/PSDで2倍規則を区別）

## 再現性
- 乱数は必要なときだけ `rng(0)` で固定する
- 章スクリプトは「Run All」で図まで一気に再現できる形にする

## 図のスタイル（暫定）
- 軸ラベルに単位を入れる（例：`Time [s]`, `Frequency [Hz]`）
- 図タイトルは「条件が分かる」ことを優先する（例：`fs=..., N=...`）
